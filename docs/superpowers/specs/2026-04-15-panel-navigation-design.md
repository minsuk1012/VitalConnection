# 썸네일 편집기 패널 네비게이션 재설계

**날짜:** 2026-04-15  
**상태:** 승인됨

---

## 목표

템플릿 브라우저와 편집 패널이 항상 동시에 표시되던 기존 레이아웃을 **단일 288px 패널**로 통합하고, 3단계 슬라이드 네비게이션으로 교체한다.

---

## 배경 및 문제점

현재 레이아웃:
```
[템플릿 브라우저 w-56=224px] + [편집 패널 w-72=288px] + [캔버스]
```

- 두 패널이 항상 동시에 노출되어 캔버스가 좁아짐
- 템플릿 선택 → 편집 전환이 명확하지 않음
- 요소 편집 시 모든 요소 컨트롤이 펼쳐져 스크롤이 길어짐
- 슬라이더가 부정확하고, 폰트 선택이 텍스트 드롭다운이라 직관성 낮음

---

## 설계

### 레이아웃 변경

```
[단일 패널 288px] + [캔버스 (기존 대비 +224px 확장)]
```

- `page.tsx`에서 `w-56` 템플릿 브라우저 aside 제거
- 편집 패널 aside를 `w-72` 유지하되 내부 구조 교체
- 캔버스 영역이 224px 넓어짐

### 3단계 네비게이션

**Level 1: 템플릿 브라우저** (기본 상태)
- 검색 인풋 + 레이아웃 필터 버튼
- 템플릿 목록 (색상 도트 + 이름 + 레이아웃 태그)
- 템플릿 클릭 → Level 2로 슬라이드 전환

**Level 2: 편집 패널**
- 상단: `← 템플릿` 뒤로가기 버튼 + 현재 템플릿명 + "편집 중" 배지
- 본문 (스크롤):
  - 언어 탭 (KO/EN/JA/ZH)
  - 레이아웃 섹션 (항상 노출)
  - 이펙트 섹션 (항상 노출)
  - 배경색 섹션 (항상 노출)
  - 요소 목록: 색상 dot + 요소명 + 현재 fontSize 힌트 + `›` 화살표 → 클릭 시 Level 3
  - 텍스트 입력 섹션 (항상 노출)
  - 저장 버튼
- 뒤로가기 → Level 1

**Level 3: 요소 컨트롤**
- 상단: `← 편집` 뒤로가기 + 색상 dot + 요소명 + "숨기기" 토글
- 타이포그래피 섹션:
  - 폰트 선택기: 버튼 클릭 시 드롭다운, 각 항목에 해당 폰트로 샘플 텍스트 표시
  - 스텝 인풋 (`[−] 값 [+]`): fontSize, lineHeight, letterSpacing
- 색상 섹션:
  - 텍스트 색상: 컬러 피커 + 8색 팔레트 퀵픽
  - 배경 색상: 동일 구조 (text 타입 요소에만)
- 위치/크기 섹션:
  - x/y: 읽기 전용 숫자 + `‹ ›` 1px 넛지 버튼 (캔버스 드래그 보조)
  - maxWidth: 스텝 인풋
  - opacity: 스텝 인풋 (%)
  - 안내 문구: "위치는 캔버스 드래그로 조정"
- 뒤로가기 → Level 2

### 슬라이드 전환

- CSS `transform: translateX()` + `transition: transform 0.15s ease-in-out`
- Level 1 → 2: L1이 `-100%`로, L2가 `100%→0`으로
- Level 2 → 3: L2가 `-100%`로, L3가 `100%→0`으로
- 역방향은 반대

### 버그 수정 (선택 즉시 해제 문제)

현재 DragCanvas에서 요소 핸들 클릭 시 `onSelectTarget(el.cssTarget)`이 설정되지만, 외부 div의 `onClick={() => onSelectTarget(null)}`이 click 이벤트 버블링으로 즉시 해제된다.

수정: 요소 핸들 div에 `onClick={e => e.stopPropagation()}` 추가.
